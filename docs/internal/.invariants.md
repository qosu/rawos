- I-SEC1 (no-lockout floor): every perimeter/SSH/LSM/firewall change has deadman timed auto-revert + session-2 + frontdoor + Hetzner Rescue confirmed before apply. Reboot-class = out-of-band confirmed first.
- I-SEC2 (least-privilege exec): user-agent code NEVER runs host-direct, NEVER root-in-container, minimum capabilities only. run_bash host-direct fails-closed for untrusted path.
- I-SEC3 (secret confinement): no high-value secret (Stripe/JWT/NIM/HF/Telegram) present in env/fs readable by sandbox process. Secrets injected only into rawos.service via systemd LoadCredential.
- I-SEC4 (tenant boundary): every context/memory/file query scoped by (user_id, project_id); no cross-tenant path. Regression tests cover all seams.
- I-SEC5 (provenance separation): TRUSTED context (constitution/system-prompt, rawos-owned) is structurally separated from UNTRUSTED (chat, tool output, fetched, perceived). Untrusted NEVER interpreted as instruction.
- I-SEC6 (capability mediation): every privileged/irreversible action flows through ONE mediated gate (graduation + floor + reversibility + audit-chain write). Untrusted-origin reasoning CANNOT mint high-tier capability. Highest tier = human-gated forever.
- I-SEC7 (tamper-evidence): every privileged action + agent decision + auth event written append-only, hash-chained, signed by key not reachable by sandbox, mirrored off-box. Chain divergence → owner alarm.
- I-SEC8 (SSRF deny): _fetch_url default-deny RFC1918 + link-local (169.254.x.x) + loopback + non-http(s); fail-closed.
- I-SEC9 (supply-chain verify): self-reload + unit authorship + dep install verify integrity (signed commit / pinned hash) BEFORE execute. Mismatch = refuse.
- I-SEC10 (rotate-on-exposure): any secret ever in plaintext = treat as exposed → rotate values, not just move.
- I-SEC11 (audit-first): every new enforcement ships DORMANT or AUDIT; only enforce after audit-period review proves no false-positives. Mirror Phase 24B discipline.
